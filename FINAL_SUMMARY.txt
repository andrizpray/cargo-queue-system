================================================================================
                      BROADCAST EVENTS
================================================================================

EVENT 1: QueueCreated
  Trigger: POST /api/queues
  Channels: queues.all, queues.{location_id}
  Event Name: queue.created
  Data: id, queue_number, location_id, vehicle_id, status, arrived_at, vehicle, location

EVENT 2: QueueStatusChanged
  Trigger: PATCH /api/queues/{id}/status
  Channels: queues.all, queues.{location_id}
  Event Name: queue.status-changed
  Data: id, queue_number, location_id, vehicle_id, previous_status, new_status, 
        started_at, completed_at, notes, vehicle, location

EVENT 3: QueueDeleted
  Trigger: Queue deletion
  Channels: queues.all, queues.{location_id}
  Event Name: queue.deleted
  Data: id, queue_number, location_id

================================================================================
                    QUICK START COMMANDS
================================================================================

Start WebSocket Server:
  php artisan reverb:start

Test with wscat:
  npm install -g wscat
  wscat -c ws://localhost:8080/app/cargo-queue-key
  {"event":"subscribe","data":{"channel":"queues.all"}}

Create Queue (triggers QueueCreated event):
  curl -X POST http://localhost:8080/api/queues \
    -H "Content-Type: application/json" \
    -d '{"vehicle_type_id":1,"location_id":1,"plate_number":"ABC-1234"}'

Update Queue Status (triggers QueueStatusChanged event):
  curl -X PATCH http://localhost:8080/api/queues/1/status \
    -H "Content-Type: application/json" \
    -d '{"status":"loading","notes":"Starting cargo loading"}'

Run Tests:
  php artisan test tests/WebSocketTest.php
